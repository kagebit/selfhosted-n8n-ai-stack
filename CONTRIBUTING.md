# Contributing / Cómo contribuir

---

## 🇬🇧 English

Thank you for your interest in contributing to **selfhosted-n8n-ai-stack**!

### Branches

- **`developer`** — active development branch. All PRs should target this branch.
- **`main`** — stable releases only.

### How to contribute

1. **Fork** this repository and clone your fork.
2. Create a new branch from `developer`:
   ```bash
   git checkout -b feat/your-feature-name
   ```
3. Make your changes and commit using the conventional commit style (see below).
4. Open a **Pull Request** targeting the `developer` branch.
5. Describe what your PR does and why in the PR description.

### Commit style

We follow [Conventional Commits](https://www.conventionalcommits.org/):

| Prefix | Use for |
|--------|---------|
| `feat:` | New features |
| `fix:` | Bug fixes |
| `docs:` | Documentation changes |
| `chore:` | Maintenance (deps, config) |
| `refactor:` | Code restructure without behaviour change |

Example: `feat: add Ollama service integration`

### Reporting issues

Open an [Issue](https://github.com/Hamza-Cloud-DevOPS/selfhosted-n8n-ai-stack/issues) and describe:
- What you expected to happen
- What actually happened
- Your OS and hardware (RAM, GPU if relevant)

## 🇪🇸 Español

¡Gracias por tu interés en contribuir a **selfhosted-n8n-ai-stack**!

### Ramas

- **`developer`** — rama de desarrollo activo. Todos los PRs deben apuntar a esta rama.
- **`main`** — solo versiones estables.

### Cómo contribuir

1. Haz un **fork** del repositorio y clónalo.
2. Crea una rama nueva desde `developer`:
   ```bash
   git checkout -b feat/nombre-de-tu-cambio
   ```
3. Aplica tus cambios y haz commit siguiendo el estilo de commits indicado más abajo.
4. Abre un **Pull Request** apuntando a la rama `developer`.
5. Describe en el PR qué hace tu cambio y por qué.

### Estilo de commits

Seguimos [Conventional Commits](https://www.conventionalcommits.org/):

| Prefijo | Usado para |
|---------|-----------|
| `feat:` | Nuevas funcionalidades |
| `fix:` | Corrección de errores |
| `docs:` | Cambios en documentación |
| `chore:` | Mantenimiento (dependencias, configuración) |
| `refactor:` | Reestructuración de código sin cambio de comportamiento |

Ejemplo: `feat: añadir integración con Ollama`

### Reportar problemas

Abre un [Issue](https://github.com/Hamza-Cloud-DevOPS/selfhosted-n8n-ai-stack/issues) e indica:
- Qué esperabas que ocurriera
- Qué ocurrió realmente
- Tu sistema operativo y hardware (RAM, GPU si aplica)
